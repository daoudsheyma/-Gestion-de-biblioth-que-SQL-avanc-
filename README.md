
Projet de conception et d’implémentation d’une base de données relationnelle pour gérer les livres, les étudiants et les emprunts, avec exploitation analytique des données et suivi des transactions.

⚠️ Pour des raisons pratiques, ce dépôt contient uniquement le code SQL (création des tables et requêtes). Le dataset n’est pas inclus.

//Ce projet réalisé dans le cadre d’un partiel SQL avancé consiste à :

Concevoir un schéma relationnel complet avec tables, clés primaires et étrangères, contraintes d’intégrité et relations.
Développer des requêtes SQL avancées pour :
Suivi des emprunts et retours 📖
Analyse des étudiants les plus actifs 👩‍🎓
Détection des livres non empruntés et des livres les plus populaires 📊
Mettre en place un système de gestion des ventes et revenus des livres 💰
Exploiter les jointures, agrégations, fonctions analytiques et sous-requêtes pour répondre à des problématiques métier concrètes.

//Technologies utilisées :

SQL / MySQL / T-SQL
Modélisation relationnelle
Analyse des données et reporting

//Compétences développées :

Data modeling et conception de bases de données
Manipulation et analyse de données relationnelles
Optimisation de requêtes SQL (GROUP BY, HAVING, fonctions analytiques)
Logique Data Engineering pour le suivi des flux d’emprunts et ventes
